# Demo Template

Use this template as a starting point for new Agent Mode demonstrations.

## Required Files

Each demo should include:

### 1. README.md
- Project overview
- Quick start guide
- Key features
- Screenshots/GIFs if applicable

### 2. DEMO.md
- Step-by-step demonstration instructions
- Exact prompts to use with Agent Mode
- Expected outcomes
- Time estimates for each phase

### 3. REQUIREMENTS.md
- Business/project requirements
- User stories
- Acceptance criteria
- Technical specifications

### 4. IMPLEMENTATION.md
- Task breakdown
- Progress tracking
- Technical decisions
- Architecture notes

### 5. CHANGELOG.md
- Version history
- Major changes
- Release notes

### 6. LICENSE
- MIT License or appropriate license file

## Naming Conventions

- **Folders**: Use kebab-case (e.g., `my-demo-name`)
- **Files**: Use PascalCase for docs (e.g., `README.md`)
- **Code**: Follow language conventions

## Tips for Great Demos

1. **Start Simple**: Begin with a clear, achievable goal
2. **Show MCP Integration**: Demonstrate at least one MCP tool
3. **Document Everything**: Record all prompts and outcomes
4. **Make it Visual**: Include screenshots or recordings
5. **Test Thoroughly**: Run through the demo multiple times
6. **Keep it Focused**: One main concept per demo
7. **Add Personality**: Use themes or creative elements
