Session Promise

We will explore Copilot Agent Mode as an autonomous pair programmer for multi-step tasks: analyzing context, generating plans, modifying code, running validation, and iteratively improving when errors appear.

Part 1: Conceptual Frame

Before the demo, build the mental model

First we explain when Agent Mode matters, how it differs from inline suggestions and chat, and which building blocks make it useful in real repositories.

Core Message

Inline Copilot helps with the next line. Agent Mode helps with the next outcome.

The Shift

From code completion to engineering workflow

InlineGreat for local code, APIs, expressions, and boilerplate.
ChatGreat for explanations, alternatives, and lightweight Q&A.
Plan ModeUseful when the task needs context, constraints, and sequencing before edits.
Agent ModeUseful when the work spans files, tests, tools, validation, and handoff.

Mental Model

Give the agent four things

ContextRepo files, requirements, tests, and prior decisions.
ConstraintsInstructions, scope, acceptance criteria, and safety boundaries.
ToolsTerminal, tests, MCP servers, browser automation, docs, or work items.
ValidationCommands, checks, review handoff, and human approval.

Part 2: Live Demo

Now we move from concept to action

The live demo uses a small release-readiness codebase so Agent Mode can plan, coordinate a council, change code, run tests, optionally use MCP, and produce a reviewable handoff.

Demo Scenario

Launch Readiness Command Center

A checkout release has mixed signals: tests are mostly green, a production incident is open, EU privacy review is unresolved, and the team needs a ship / hold / limited-rollout recommendation.

What The Repo Contains

Codesrc/readiness.js, src/signals.js, tests/readiness.test.js
InstructionsAGENTS.md tells Agent Mode how to work in this demo.
PromptsReusable flows for planning, council, implementation, and MCP moments.
SkillsDomain knowledge for release readiness, agent councils, MCP showcase, and storytelling.

Live Flow

1Show inline baseline in ordinary JavaScript.
2Use Plan Mode to scope the release-readiness change.
3Run an Agent Council with specialized roles.
4Let Agent Mode edit code and update tests.
5Validate, add an MCP moment, and hand off like a PR.

Agent Council

One chat, multiple perspectives

PlannerFrames scope, assumptions, and validation.
Release StrategistOwns ship, hold, or limited rollout.
QA AnalystChecks tests, flakes, and regression risk.
Security ReviewerChecks privacy and security blockers.

Prompt 1

Start with Plan Mode

Read AGENTS.md and requirements/launch-request.md. Use Plan Mode. Create a small implementation plan, identify risks, and name the cheapest validation command. Do not edit files.

Prompt 2

Run the council in chat

Run an Agent Council for this change. Use Planner, Release Strategist, QA Analyst, Security Reviewer, and Implementer perspectives. Synthesize one recommendation instead of giving five disconnected reports.

Prompt 3

Let Agent Mode implement

Implement the EU privacy-review hard blocker. Keep the change focused. Update tests. Run validation. Stop and report the result.

MCP Moment

Tools connect the agent to real engineering systems

GitHubIssues, PR context, review handoff.
Microsoft LearnOfficial docs for Q&A and grounding.
PlaywrightBrowser automation and runtime validation.
Azure / Work ItemsDeployment context and acceptance criteria.

Takeaway

Agent Mode is not a bigger autocomplete.

It is a way to give Copilot context, constraints, tools, roles, validation, and a handoff discipline.
